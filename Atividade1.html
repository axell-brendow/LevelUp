- As 3 camadas da interface: Composição: é a camada que faz um arranjo agradável dos conteúdos da aplicação. É também onde entram de forma mais efetiva os elementos de manipulação da interface como por exemplo a linguagem JavaScript e o CSS. Conteúdo: é a camada onde fica o conteúdo de fato. Todos os textos, imagens, vídeos e demais informações estão inseridos nessa camada. E é muito importante que a Camada de Controle e Navegação leve o usuário ao conteúdo. Controle e Navegação: é a camada que está relacionada ao acesso ao conteúdo. Também é onde o usuário terá as primeiras dificuldades para navegar pelos conteúdos e consumi-los. Sendo assim, é a parte crucial da interface por questões de usabilidade. - Por que o sistema de metáforas pode ser restritivo para alguns usuários ? Pode ser restritivo por não usar a metáfora certa para certos tipos de usuários. Um exemplo: o famoso botão de hamburguer das interfaces. O botão que abre os menus (3 barras horizontais). Conheço vários usuários mais velhos que nunca pensaram em tocar nele pois ele não os atraiu. Acredito que a analogia fosse simular algo parecido como uma lista de opções, por isso as 3 barras, mas não foi muito efetivo. Isso ainda é usado por ser um padrão. - Qual a importância da arquitetura de informação para um projeto de Front-end? Ela traz uma discussão sobre os 3 pilares das interfaces: conteúdo, usuário e contexto. Além disso, ela mostra os passos para: organizar o conteúdo, verificar se essa organização está boa para cada usuário, criar modelos genéricos de organização, criar fluxos de tarefas, definir as estruturas de navegação e realizar testes com protótipos. Fala também sobre os seguintes conceitos: Taxonomia: categorizar o conteúdo. Terminologia: cada contexto tem os termos adequados para apresentar o conteúdo. Ontologia: formas de representar o conhecimento em diferentes formas. Recuperação de informação e sistemas de navegação: como o usuário irá chegar à informação (recuperá-la). Quebra de conteúdo e unidades lógicas: importante principalmente quando um tópico é muito grande e é necessário quebrá-lo em unidades menores. - Explique a diferença de layout Líquido e Responsivo. O que difere os dois é a forma como se adaptam para resoluções diferentes. Layout líquido (ou fluído): a adaptação é mais baseada em porcentagens. A medida que a tela diminui, as divisões diminuem proporcionalmente. Layout responsivo: agrega também um pouco do layout líquido, porém, em alguns casos, invés de simplesmente diminuir as divisões proporcionalmente, altera a forma como elas estão dispostas na interface. - Construa um exemplo de taxonomia que utilize navegação global, local e contextual. Navegação global: links para as categorias do primeiro nível do site e menus de navegação que estão em todas as páginas. Navegação local: menus específicos para navegar em subáreas e seções do site. Navegação contextual: links relacionados ao conteúdo principal que está sendo apresentado.
